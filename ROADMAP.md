# Research AI Lab Roadmap

## Guiding Logic

Research AI Lab is evolving in layers. Each release should add durable capabilities without weakening coherence, quality, or governance.

## Version 1.0

Focus: Foundation architecture

- Constitution and manifesto
- Strategic identity
- Documentation standards
- Repository architecture
- Toolkit specification
- Economics Lab

Release criteria:

- All foundational documents published
- README communicates the initiative clearly
- Economics Lab demonstrates the methodology end to end
- Contribution and governance model are understandable

## Version 1.1

Focus: Education Lab

- New canonical lab focused on education research
- Adaptation of toolkit patterns for education workflows
- Additional examples and teaching-oriented resources

Release criteria:

- Education Lab follows RAL standards
- Clear differentiation between lab-neutral and domain-specific methods
- At least one reusable education workflow example

## Version 1.2

Focus: Artificial Intelligence Lab

- Lab on AI research itself
- Stronger guidance on evaluating model claims, benchmarks, and agentic workflows
- Expanded prompt and evidence review resources

Release criteria:

- AI Lab introduces domain-relevant methods responsibly
- Benchmark and claim-evaluation guidance is explicit
- Vendor-neutral framing preserved

## Version 2.0

Focus: Research Agents

- Agent specifications for bounded research tasks
- Guardrails for traceability and human review
- Reusable agent patterns aligned with RAL methodology

Release criteria:

- Agent workflows map clearly to documented research stages
- Human-in-the-loop requirements are explicit
- Reproducibility and auditability remain intact

## Version 3.0

Focus: Research Operating System

- Integrated environment for labs, toolkit, workflows, references, and agents
- Cross-lab orchestration patterns
- Shared evidence and documentation infrastructure

Release criteria:

- Architecture supports multi-lab scaling
- Governance and quality systems still function under growth
- Community contributions can extend the system without fragmenting it
