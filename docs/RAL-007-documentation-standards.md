# RAL-007: Documentation Standards

## Purpose

This document defines how future Research AI Lab documents should be written.

## Writing Style

- Professional
- Clear
- Concise
- Evidence aware
- Accessible across disciplines

## Structure

Every substantial document should include:

- Clear title
- Purpose or context
- Well-defined sections
- Explicit assumptions where relevant
- Conclusion or actionable takeaway when appropriate

## Naming

- Use stable identifiers for normative documents
- Use descriptive filenames
- Avoid ambiguous abbreviations unless already standardized

## Versioning

- Major documents should carry version-aware context through changelog or release notes
- Structural revisions should be traceable

## Metadata

Where useful in future phases, documents may include:

- Status
- Version
- Owner or maintainer
- Last updated date
- Related documents

## References and Bibliography

- Substantive claims should be sourced where appropriate
- Distinguish between primary references and supporting reading
- Prefer stable links and durable citations

## Citations

- Keep citation style consistent inside a given document
- Prefer enough detail for a reader to locate the source
- Avoid untraceable claims presented as settled fact

## Quality Checklist

Before publication, confirm that the document:

- Supports the mission and principles
- Uses clear structure
- Avoids unnecessary jargon
- States limitations where needed
- Can still make sense if tools change
- Contributes to a coherent architecture
