# Contributing to Research AI Lab

Thank you for helping build Research AI Lab.

This initiative is designed as open methodological infrastructure for AI-assisted scientific research. We welcome thoughtful contributions that improve rigor, clarity, reproducibility, accessibility, and community usefulness.

## What Good Contributions Look Like

Strong contributions usually do one or more of the following:

- Clarify or strengthen methodological guidance
- Improve documentation structure or readability
- Add reproducible examples
- Expand toolkit components without locking the project to a single vendor
- Propose new labs grounded in meaningful research questions
- Improve transparency, citation practice, or evidence quality

## Contribution Principles

- Methodology before novelty
- Clarity before complexity
- Evidence before opinion
- Openness before gatekeeping
- Reproducibility before convenience
- Community value before personal branding

## Ways To Contribute

- Open an issue to suggest an improvement
- Propose edits to a document
- Add examples or templates
- Expand the toolkit
- Improve the Economics Lab methodology
- Propose future labs aligned with the roadmap
- Help translate or localize materials

## Before You Submit

Please make sure your contribution:

- Aligns with the mission and principles in [`docs/RAL-000-constitution.md`](docs/RAL-000-constitution.md)
- Follows the standards in [`docs/RAL-007-documentation-standards.md`](docs/RAL-007-documentation-standards.md)
- Is understandable by readers beyond your immediate specialty
- Preserves vendor neutrality unless a tool-specific example explicitly requires otherwise
- Includes references when making substantive claims

## Pull Request Expectations

Every pull request should describe:

- What changed
- Why it matters
- Which document, workflow, or lab it affects
- Any assumptions or tradeoffs
- Whether the change is normative, exploratory, or editorial

## Style Expectations

- Write in a precise, accessible, professional tone
- Prefer plain language over jargon when possible
- Use headings and structure generously
- Keep claims traceable to sources
- Distinguish recommendations from requirements

## Review Criteria

Contributions are reviewed for:

- Mission alignment
- Methodological quality
- Internal consistency
- Evidence awareness
- Documentation quality
- Reusability

## Code of Conduct

By participating, you agree to follow [`CODE_OF_CONDUCT.md`](CODE_OF_CONDUCT.md).
